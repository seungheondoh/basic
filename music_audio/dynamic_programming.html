Audio Matching with Dynamic Programming (다이나믹 프로그래밍을 이용한 오디오 매칭)#

동적 프로그래밍(Dynamic Programming, DP)은 복잡한 문제를 간단한 하위 문제로 나누어 효율적으로 해결하는 알고리즘 설계 기법입니다. 이 방법은 하위 문제의 해답을 저장하고 재사용함으로써 중복 계산을 피하고 성능을 개선합니다.

  • 동적 프로그래밍에서는 매번 현재의 최대 가치를 저장해놓고 비교해보기 때문에 답이 이전의 값보다 더 나빠지는 경우는 있을 수 없습니다.

  • 동적 프로그래밍은 각 하위 문제들이 서로 관계가 없을 때, 즉 서로 의존하지 않는 경우에만 쓸 수 있습니다.

  • 동적 프로그래밍은 제한 조건이 있는 경우에 무언가를 최적화할 때 유용합니다.

  • 모든 동적 프로그래밍 알고리즘은 격자grid로부터 시작합니다. 격자의 각 칸은 하위 문제를 뜻합니다. 그러므로 원래의 문제를 어떻게 하위 문제를 나눌 수 있는지 생각해야 합니다.

Memoization#

Memoization는 동적 프로그래밍에서 사용되는 캐싱입니다. 캐싱의 목적은 나중에 사용할수 있는 데이터를 접근 가능하게 유지하는 것입니다. 기본적으로 이전에 계산된 하위 문제의 결과를 저장하고 동일한 하위 문제에 대해서 저장된 결과를 사용합니다. 이는 동일한 문제에 대해서 반복 계산을 하게 되는 문제를 없애줍니다.

DP의 기본 구조#

  • 하위 문제 정의: 큰 문제를 해결할 수 있는 작은 하위 문제로 나눕니다.

  • 재귀적 구조: 각 하위 문제는 동일한 방식으로 더 작은 하위 문제로 나뉩니다.

  • 하위 문제 해결 저장: 각 하위 문제의 해결책은 한 번 계산되면 저장됩니다.

  • 하위 문제 재사용: 저장된 하위 문제의 해결책을 필요할 때 재사용하여 중복 계산을 방지합니다.

Template based Audio Matching#

때문에 ASR에서는 음성 신호(시간에 따른 연속적인 오디오 데이터)와 해당 신호가 나타내는 언어 단위(예: 음절, 단어) 간의 최적의 매칭을 찾아야 합니다. 동적 프로그래밍은 이러한 매칭 문제를 효과적으로 해결하는 데 사용됩니다. 특히, Dynamic Time Warping(DTW) 같은 알고리즘이 이 과정에서 중요합니다.

Dynamic Time Warping (DTW)은 시간에 따른 두 시계열 데이터 간의 유사도를 측정하는 알고리즘입니다. 이 기법은 특히 음성 인식과 같이 데이터가 시간에 따라 늘어나거나 줄어들 수 있는 상황에서 유용합니다. DTW는 두 시계열 데이터가 서로 다른 속도로 진행되더라도 그들 사이의 최적 매칭을 찾아내는 데 중점을 둡니다.

Audio-Audio Alignment#

사람들은 같은 단어나 문장을 말할 때 속도나 강조가 다를 수 있습니다. Dynamic Programming을 사용하는 Alignment 알고리즘은 이러한 시간적 변동성을 효과적으로 처리하여, 다양한 발음 속도나 스타일에도 정확한 인식을 가능하게 합니다. Template 기반 ASR은 기본적으로 미리 정의된 단어나 구문의 오디오 템플릿을 사용하여 입력된 음성을 인식합니다. 이 과정에서 DP는 중요한 역할을 합니다:

  • Template Matching: DP는 입력된 음성과 저장된 템플릿 간의 최적의 매칭을 찾는 데 사용됩니다. 이는 입력 음성과 템플릿 간의 시간적 변동성을 조정하고, 가장 유사한 템플릿을 효과적으로 찾는 데 도움을 줍니다.

  • 시간 왜곡(Time Warping): 사람들의 발음은 속도와 리듬이 다를 수 있어, 동일한 단어나 문구라도 오디오 신호가 다르게 나타날 수 있습니다. DP는 이러한 시간적 차이를 조정하는 데 사용되어, 불일치를 최소화합니다.

import torch
import torchaudio
import numpy as np
from torchaudio.transforms import MFCC

# MFCC 변환 함수 초기화
mfcc_fn = MFCC(sample_rate=sample_rate, n_mfcc=13)

def euclidean_distance(vec1, vec2):
    return np.linalg.norm(vec1 - vec2)

def dtw_mfcc_with_path(mfcc1, mfcc2):
    n, m = mfcc1.shape[1], mfcc2.shape[1]  # 시간 축의 길이를 사용
    dtw_matrix = np.full((n + 1, m + 1), float('inf'))
    dtw_matrix[0, 0] = 0

    # 두 mfcc frame을 for loop로 조회
    for i in range(1, n + 1):
        for j in range(1, m + 1):
            cost = euclidean_distance(mfcc1[:, i - 1], mfcc2[:, j - 1]) # mfcc euclidean_distance == cost
            # Recursive 파트는 여기!
            dtw_matrix[i, j] = cost + min(dtw_matrix[i - 1, j],    # 삽입
                                          dtw_matrix[i, j - 1],    # 삭제
                                          dtw_matrix[i - 1, j - 1])  # 매치
    # DTW 경로 추적
    path = []
    i, j = n, m
    while i > 0 and j > 0:
        path.append((i - 1, j - 1))
        min_cost = min(dtw_matrix[i - 1, j], dtw_matrix[i, j - 1], dtw_matrix[i - 1, j - 1])
        if min_cost == dtw_matrix[i - 1, j]:
            i -= 1
        elif min_cost == dtw_matrix[i, j - 1]:
            j -= 1
        else:
            i -= 1
            j -= 1
    path.reverse()
    return dtw_matrix, dtw_matrix[n, m], path
